## Key Differences from VS Code

| Feature | VS Code | BMad Studio |
|---------|---------|-------------|
| Extension context header | "VSCode native extension environment" | "BMad Studio, a web-based IDE" |
| Code reference format | Markdown links `[file.ts](path)` | Markdown links `[file.ts](path)` (identical) |
| User selection context | IDE selection via `ide_selection` tags | Not available (no IDE integration yet) |
| Git status | Full `git status` output | `git status --short` (truncated at 30 lines) |
| Git status description | Snapshot notice included | Snapshot notice included (identical) |
| Main branch format | "Main branch (you will usually use this for PRs):" | Identical |
| Main branch detection | Provided by VS Code | Auto-detected via `git rev-parse --verify` |
| Additional directories | Via VS Code workspace folders | Not supported yet |
| Total appended tokens | ~3,500 tokens | ~2,500-3,500 tokens (depends on git status size) |

### What was missing before this fix (caused path hallucination):

Before adding `buildWorkspaceContext()`, the BMad Studio system prompt had **NONE** of the
appended context. The SDK base prompt includes `Primary working directory: D:\repo\private\hello`
in the Environment section, but without additional file path grounding (git status, directory
listing), the model would hallucinate paths like `/Users/jake/test.txt`.

### Token comparison:
- VS Code total system prompt: ~19,876 tokens
- BMad Studio before fix: ~16,403 tokens (missing ~3,473 tokens of context)
- BMad Studio after fix: ~17,000-18,000 tokens (with workspace context appended)
