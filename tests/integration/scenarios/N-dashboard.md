# N. 대시보드 실시간 상태

**범위**: 프로젝트 리스트 상단 5개 카드, 구독/구독해제 라이프사이클.
**선행 도메인**: A.

---

## N1. 5개 통계 카드 `[CORE] [ASYNC]`

### N-01-01: 초기 렌더 값 검증
**절차**: `/projects` 진입.
**기대 결과**:
- 카드: 프로젝트 수 / 세션 수 / 활성 세션 / 큐 실행 / 터미널
- 각 수치가 실제 서버 상태와 일치 (e.g. 세션 리스트 합과 동일)

### N-01-02: 실시간 증감
**절차**:
1. 탭 A: 대시보드 가시
2. 탭 B: 새 세션 생성 + 메시지 스트리밍 시작
3. 탭 A 관찰

**기대 결과**:
- "세션 수" +1 (300ms 디바운스 이후)
- "활성 세션" +1 (스트리밍 중)
- 스트리밍 종료 시 "활성 세션" -1

---

## N2. 구독 / 구독해제 라이프사이클 `[EDGE]`

### N-02-01: 다른 페이지로 이동 시 구독 해제
**절차**:
1. 대시보드 진입 (`dashboard:subscribe` 발송 확인)
2. 프로젝트 페이지로 이동 (`dashboard:unsubscribe` 확인)

**기대 결과**: 불필요 이벤트 스트림 중지, 서버 리소스 정리.

### N-02-02: 다중 브라우저 동시 구독
**절차**: 동일 계정 두 브라우저에서 동시에 대시보드.
**기대 결과**: 둘 다 동일 상태 수신, 서버 중복 브로드캐스트 과다 없음.

### N-02-03: 장시간 유지
**기대 결과**: WebSocket ping/pong으로 연결 유지, 재연결 시 자동 재구독.
