# J. 파일 탐색기

**범위**: 트리/그리드 탐색, 편집 & 저장, CRUD, 파일명 검색.
**선행 도메인**: A, B. 뷰어 상세는 S 도메인 참조.

---

## J1. 트리 / 그리드 뷰 `[CORE]`

### J-01-01: 디렉토리 탐색
**절차**: 파일 탐색기 탭 → 루트에서 하위 폴더 토글.
**기대 결과**: 확장/축소 상태 유지, 하위 파일 표시.

### J-01-02: 그리드 뷰
**기대 결과**: Finder 스타일 아이콘, 파일명 하단 표시, 대용량 폴더(1000+)에서도 가상 스크롤.

---

## J2. 파일 편집 & 저장 `[CORE]`

### J-02-01: 텍스트 파일 열기·수정·Ctrl+S
**절차**:
1. `.txt` 파일 클릭 → TextEditor(CodeMirror) 열림
2. 내용 수정 → 제목바에 "*" 표시
3. `browser_press_key("Control+s")`

**기대 결과**:
- 저장 성공 → "*" 제거
- 디스크 파일 실제 변경 (서버 측 확인)

### J-02-02: 코드 파일 문법 하이라이트
**기대 결과**: 언어별 토큰 색상(CodeMirror), 에러 표시 없음.

### J-02-03: 바이너리 감지 (→ S5 참조)
**기대 결과**: 편집기 대신 "Download" 버튼 표시.

---

## J4. 파일 CRUD · 외부 변경 감지 `[EDGE]`

### J-04-01: 새 파일 / 폴더 생성
**절차**: 우측 클릭 → "New File" → 이름 입력.
**기대 결과**: 트리 즉시 갱신.

### J-04-02: 이름 변경 / 삭제
**기대 결과**: 삭제 확인 모달, 실행 후 디스크에서 제거.

### J-04-03: 외부 프로세스 변경 감지
**절차**: 탐색기에서 파일을 열어둔 상태로 외부 프로그램/터미널에서 동일 파일 수정.
**기대 결과**: 에디터에 외부 변경 감지 알림, 리로드/병합 옵션 제공.

---

## J5. 파일명 검색 `[CORE]`

### J-05-01: searchFiles API 호출
**절차**: 상단 검색창에 키워드 입력.
**기대 결과**:
- 매치 파일 리스트 표시
- `.gitignore` 대상 기본 제외
- 대용량 프로젝트에서 2초 내 응답 (네트워크 확인)

**엣지케이스**:
